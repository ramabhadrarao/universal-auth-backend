Principles Module API - cURL Examples

Note: Replace YOUR_JWT_TOKEN with a valid JWT token obtained through login. Replace IDs (like PRINCIPLE_ID) with actual IDs from your database.

Main Principle Endpoints

GET Get All Principles

Endpoint: /api/v1/principles

curl -X GET "http://localhost:5000/api/v1/principles?page=1&limit=10" \ -H "Authorization: Bearer YOUR_JWT_TOKEN"

GET Get Single Principle

Endpoint: /api/v1/principles/:id

curl -X GET http://localhost:5000/api/v1/principles/PRINCIPLE_ID \ -H "Authorization: Bearer YOUR_JWT_TOKEN"

POST Create New Principle

Endpoint: /api/v1/principles

curl -X POST http://localhost:5000/api/v1/principles \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "name": "ABC Medical Supplies", "contactPerson": "John Smith", "email": "contact@abcmedical.com", "phone": "1234567890", "address": "123 Medical Plaza, City", "website": "https://abcmedical.com", "gstNumber": "GST123456", "paymentTerms": "Net 30", "creditDays": 30 }'

PUT Update Principle

Endpoint: /api/v1/principles/:id

curl -X PUT http://localhost:5000/api/v1/principles/PRINCIPLE_ID \ -H "Authorization: Bearer YOUR_JWT_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "contactPerson": "Jane Smith", "email": "jane@abcmedical.com", "phone": "9876543210" }'

DELETE Delete Principle

Endpoint: /api/v1/principles/:id

curl -X DELETE http://localhost:5000/api/v1/principles/PRINCIPLE_ID \ -H "Authorization: Bearer YOUR_JWT_TOKEN"
