Hospital API Documentation

This documentation provides details about the Hospital API endpoints, including request parameters, headers, and example curl commands.

Note: For all authenticated endpoints, include the Authorization header with a valid JWT token. In the examples below, replace {YOUR_TOKEN} with your actual token, and replace {BASE_URL} with your API base URL (e.g., http://localhost:5000).

Hospital Management

GET /api/v1/hospitals

Get all hospitals with optional filtering and pagination

Query Parameters:

Parameter Type Description
city String Filter hospitals by city (case-insensitive partial match)
state String Filter hospitals by state (case-insensitive partial match)
isActive Boolean Filter hospitals by active status ('true' or 'false')
page Number Page number for pagination (default: 1)
limit Number Results per page for pagination (default: 10)

Example curl command:

curl -X GET "{BASE_URL}/api/v1/hospitals?city=Mumbai&state=Maharashtra&isActive=true&page=1&limit=10" \
