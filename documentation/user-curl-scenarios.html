User Management cURL Scenarios

Scenario Description cURL Command
Create User Create a new user by admin 
curl -X POST http://localhost:5000/api/v1/users \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "name": "New User",
    "email": "newuser@example.com",
    "password": "Password123!",
    "status": "active",
    "roles": ["user"]
}'
Get All Users Retrieve list of users with pagination 
curl -X GET "http://localhost:5000/api/v1/users?page=1&limit=10" \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Get Single User Retrieve details of a specific user 
curl -X GET http://localhost:5000/api/v1/users/USER_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Update User Update user details 
curl -X PUT http://localhost:5000/api/v1/users/USER_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "name": "Updated Name",
    "email": "updated@example.com",
    "roles": ["admin"]
}'
Update User Status Change user account status 
curl -X PUT http://localhost:5000/api/v1/users/USER_ID/status \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "status": "suspended"
}'
Delete User Remove a user account 
curl -X DELETE http://localhost:5000/api/v1/users/USER_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Get User Statistics Retrieve user count by status 
curl -X GET http://localhost:5000/api/v1/users/stats \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
