Role Management cURL Scenarios

Scenario Description cURL Command
Create Role Create a new role with permissions 
curl -X POST http://localhost:5000/api/v1/roles \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "name": "editor",
    "description": "Content editor role",
    "permissions": ["PERMISSION_ID_1", "PERMISSION_ID_2"]
}'
Get All Roles Retrieve list of all roles 
curl -X GET http://localhost:5000/api/v1/roles \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Get Single Role Retrieve details of a specific role 
curl -X GET http://localhost:5000/api/v1/roles/ROLE_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Update Role Update role details 
curl -X PUT http://localhost:5000/api/v1/roles/ROLE_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "name": "updated_editor",
    "permissions": ["NEW_PERMISSION_ID_1", "NEW_PERMISSION_ID_2"]
}'
Delete Role Remove a role 
curl -X DELETE http://localhost:5000/api/v1/roles/ROLE_ID \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Get Users with Role Retrieve users assigned to a specific role 
curl -X GET http://localhost:5000/api/v1/roles/ROLE_ID/users \
-H "Authorization: Bearer ADMIN_JWT_TOKEN"
Add Permissions to Role Add new permissions to an existing role 
curl -X POST http://localhost:5000/api/v1/roles/ROLE_ID/permissions \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "permissions": ["PERMISSION_ID_3", "PERMISSION_ID_4"]
}'
Remove Permissions from Role Remove specific permissions from a role 
curl -X DELETE http://localhost:5000/api/v1/roles/ROLE_ID/permissions \
-H "Authorization: Bearer ADMIN_JWT_TOKEN" \
-H "Content-Type: application/json" \
-d '{
    "permissions": ["PERMISSION_ID_1", "PERMISSION_ID_2"]
}'
